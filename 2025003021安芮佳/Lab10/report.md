# Lab10 实验报告：为 Lunch Tray 添加导航
## 1. Compose Navigation 核心组件关系
- **NavController**：导航的核心控制组件，负责管理页面返回栈、执行页面跳转、处理返回逻辑。通过 `rememberNavController()` 创建，与 Composable 生命周期绑定，保证配置变更后状态不丢失，是连接页面与导航逻辑的桥梁。
- **NavHost**：导航容器组件，作为所有可导航页面的载体，接收 NavController 并配置导航规则。它定义了应用的整体导航结构，管理页面的显示、切换与栈状态，是导航功能的承载中心。
- **composable()**：路由注册函数，用于在 NavHost 中声明每一个页面，通过唯一的 route 字符串与页面绑定，建立“路由标识—UI组件”的映射关系，实现页面的模块化管理。

三者协同工作：**NavController 发出指令，NavHost 提供容器，composable() 定义页面规则**，共同完成稳定、类型安全的导航流程。

## 2. LunchTrayScreen 枚举类设计说明
本实验使用枚举类管理页面路由，而非直接使用字符串，主要优势如下：
1. **类型安全**：枚举值在编译期即可校验，避免手写字符串出现拼写错误，减少运行时崩溃风险。
2. **统一管理**：将页面路由、标题资源绑定在一起，便于维护、扩展和修改，符合软件工程高内聚设计思想。
3. **语义清晰**：代码可读性更强，看到 `LunchTrayScreen.Entree` 即可明确页面含义，比硬编码字符串更直观。
4. **便于扩展**：未来可新增图标、权限、动画等属性，无需修改大量业务代码，提升项目可维护性。
5. **避免魔法值**：减少散落在代码中的字符串常量，让结构更规范、更专业。

## 3. LunchTrayAppBar 设计思路
顶部导航栏是用户感知页面状态的核心组件，设计目标为**自动适配标题、智能显示返回按钮、保持风格统一**。
1. **动态标题**：根据当前路由页面自动显示对应标题，通过枚举类中的资源 ID 加载字符串资源，实现多语言支持与统一管理。
2. **返回按钮逻辑**：仅在非起始页面显示返回按钮，通过判断返回栈是否存在上一页实现，避免起始页出现无效返回按钮。
3. **返回行为**：点击返回按钮执行 `navigateUp()`，遵循系统默认返回栈逻辑，保证导航行为一致性与用户体验。
4. **样式统一**：使用 Material3 主题样式，保持与应用整体设计风格一致，提升界面美观度与专业性。

## 4. 导航流程与返回堆栈管理
本次实验点餐流程为：
**Start → Entree → SideDish → Accompaniment → Checkout**

导航规则设计：
1. **进入流程**：从 Start 点击开始订单后，跳转到 Entree，并将 Start 页面弹出返回栈，避免用户返回后回到起始页。
2. **顺序跳转**：点击 Next 依次进入下一页，保留返回栈，支持用户返回上一页修改选择。
3. **取消订单**：任意页面点击 Cancel，直接返回 Start 并清空返回栈，同时重置订单数据。
4. **提交订单**：结算完成后返回 Start，清空栈并重置状态，保证下次点餐为全新流程。

返回栈管理意义：
- 保证流程闭环，用户不会陷入无限返回循环。
- 提升体验，符合手机应用常见操作直觉。
- 避免页面重复创建，减少内存占用与状态混乱。

## 5. 实验问题与解决过程
### 问题1：Unresolved reference: R
- 原因：资源文件缺失、strings.xml 未定义对应字符串、项目未重新构建。
- 解决：检查并补全 strings.xml 资源，执行 Clean + Rebuild，重新生成 R 类。

### 问题2：Kotlin 与 Compose 版本不兼容导致编译失败
- 原因：Compose Compiler 版本与 Kotlin 版本不匹配。
- 解决：升级 `kotlinCompilerExtensionVersion` 或添加兼容忽略配置，使版本匹配。

### 问题3：找不到页面组件（如 StartOrderScreen）
- 原因：包名不一致、文件未放在正确目录、文件名与函数名不匹配。
- 解决：统一包路径、修正文件位置、重新同步项目。

### 问题4：Cancel 后订单数据未清空
- 原因：只做页面跳转，未调用 ViewModel 的重置方法。
- 解决：在 Cancel 逻辑中添加 `viewModel.resetOrder()`。

### 问题5：Start 页面错误显示返回按钮
- 原因：返回栈判断逻辑错误。
- 解决：使用 `previousBackStackEntry != null` 动态判断，而非固定页面判断。

### 问题6：Gradle 下载超时、同步失败
- 原因：网络环境限制，无法访问国外仓库。
- 解决：使用本地 Gradle、切换阿里云镜像、关闭在线依赖检查。

## 6. 实验心得与总结
本次实验通过为 Lunch Tray 应用实现完整导航功能，深入理解了 **Jetpack Compose Navigation** 的组件结构、路由管理、返回栈控制与状态同步机制。通过枚举类优化路由设计，提升了代码安全性与可维护性；通过顶部导航栏的动态适配，掌握了 Compose 中根据状态动态更新 UI 的方法。

实验过程中遇到了版本兼容、资源缺失、导航逻辑错误、构建同步失败等问题，通过逐一定位与修复，提升了 Android 项目调试、错误分析与解决实际工程问题的能力。同时理解了在移动应用开发中，合理的导航设计不仅影响代码结构，更直接决定用户体验。

本次实验为后续复杂应用的页面管理、状态维护、组件化开发打下了扎实基础。
